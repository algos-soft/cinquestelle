Struttura delle preferenze

aprile 2007

Le preferenze sono oggetti di tipo Pref raggruppati in alcune enumeration che vengono create ad avvio programma

La classe Pref mantiene alcune enumeration di uso generale; le sottoclassi di Pref mantengono enumeration di preferenze specifiche

Ogni enumeration deve implementare tutti i metodi necessari per le preferenze. Non è possibile sovrascrivere i metodi, perché le enumeration non possono avere superclassi/sottoclassi

* Parametri richiesti per costruire un oggetto preferenza (vengono mantenuti per mezzo un'istanza della classe PrefWrap): * Alias * Sigla * Descrizione * Livello * Tipo di dati * Lista di valori (solo per il combo) * Valore standard * Nota * Nota programmatore * Visibile * Mostra valore di default * prima istallazione
