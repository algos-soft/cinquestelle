Adattatori astratti per le Azioni.

Questo package contiene le classi astratte degli adattatori per ogni interfaccia di eventi implementata
L'inserimento di questo layer intermedio nella catena di ereditarietà di una classe di tipo Azione, deriva dalla necessità di specializzare le interfaccie di ogni singolo tipo di evento
Quando si aggiunge un'azione come Listener ad un oggetto della GUI, questo utilizza dei metodi add... diversi per ogni tipologia di eventi gestita. Se in questa fase si utilizzasse un'interfaccia comprensiva di tuttti i possibili tipi di eventi, l'oggetto accetterebbe l'inserimento senza dare errore, ma non gestirebbe l'evento in runtime
Avere un'interfaccia specifica per la tipologia di eventi gestita dall'oggetto GUI, permette di associare all'oggetto stesso solo gli eventi che questo è in grado di gestire

La classe AzAdapterAction è inserita nel package solo per omogeneità con gli altri Adapter. In realtà il metodo actionPerformed viene necessariamente inplementato nella classe AzioneBase che estende, tramite AbstractAction l'interfaccia Action
