Gestione delle relazioni ambigue:

Esistono due tipi di ambiguita' per andare dalla tavola A alla tavola B:

  • Tipo 1: piu' di un campo della tavola A puntano alla tavola B
    • (es. linkpiatto e linkcontorno in Ristorante.righemenupiatto, che puntano entrambi alla tavola Piatto).

  • Tipo 2: esistono piu' percorsi possibili tra le due tavole.
    • es CONTO -> CLIENTE -> FATTURA -> CONTO

    Le relazioni ambigue vengono gestite automaticamete dalla query nel modo seguente:

  • Ambiguit� di tipo 1:
    • Uno solo dei campi della tavola A viene contrassegnato nel modello come relazione preferita; la query usa tale campo per la relazione

  • Ambiguita' di tipo 2 (in ordine di priorita'):
    • Il modulo puo' contenere un elenco di passaggi obbligati che serve per risolvere le relazioni ambigue per tutte le query del modulo.

  • Se sono stati definiti dei passaggi obbligati nel modulo, la query li usa per risolvere l'ambiguit�.
  • Se dopo di questo resta un'ambiguita', se un percorso e' piu' breve di tutti gli altri, viene usato il percorso piu' breve

    • In aggiunta a questi automatismi, e' sempre possibile aggiungere dei passaggi obbligati a una query specifica per costringerla a utilizzare un determinato percorso.
